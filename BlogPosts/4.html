We know that customers strive for improving their operational efficiency. Today, we discuss Docker, a tool to build containerized applications. We outline its numerous advantages.

What is Docker?

Docker is a platform that allows developers to containerize their applications by bundling code and dependencies in a container. Developers and operations personnel can easily share, deploy, and monitor containerized apps – the host just needs to have the container runtime installed.

In practice, developers have multiple tools at their disposal to develop containerized applications. Visual Studio 2019 even automatically generates a Dockerfile to containerize .NET applications, if that option is selected when creating the project.

Just using a common development tool – Visual Studio 2019 – reveals the workflow of containers:

  • Start with a base container image (the “template” of the container; the Microsoft ASP.NET image in the Dockerfile above)
  • Bundle app code and dependencies with the image (copying app source code and restoring packages from the .csproj manifest)
  • Produce a new image using Docker (invoking the docker build command)
  • Create a container from the new image (using the docker run command)
The Power of Containerization

We touched the surface of containerization, and there are certainly more nuances to this concept.

  • To store and share images, developers use container registries, like Docker Hub or Azure Container Registry.
  • To deploy containers, there are a plethora of deployment options, just on the Microsoft stack, such as Azure Container Instances, Azure Kubernetes Service, and Azure App Service.
  • There are opportunities to automate the image build and release process using CI/CD tools, such as GitHub Actions. At Hessatec, we use GitHub Actions to automate the deployment of .NET applications for Singularity.

Here are more key benefits to consider when architecting your application using containers:

  • Efficiency. Developers have complete control over the layout of their containers. Operations personnel simply deploy containers without needing to consider how the container is configured on the inside; the container must simply render its services.
  • Flexible deployment options. Perhaps you have heard of Kubernetes. In simple terms, Kubernetes orchestrates the deployment of containers to provide highly available, scalable services. As YAML files define Kubernetes resources, teams can version them. Does your current infrastructure setup allow you to visualize your configuration one month ago? One year ago?
